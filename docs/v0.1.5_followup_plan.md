# v0.1.5 后续实现计划（前端轮询组件化 + purge 可观测回执）

承接 v0.1.4，本轮聚焦前端体验一致性与可观测性，不改后端契约。

## P1-A 前端轮询统一为 hook

### 现状
- `ImageInputPanel`、`QQImportPanel`、`TargetManager`、`StyleTestPanel` 各自重复 `cancelledRef + useEffect` 卸载取消样板。
- `DataPanel` 还有独立的 `pollBackupResult`（本地 `delay` + 轮询循环），没复用 `pollJobResult`。

### 方案
- 新增 `frontend/src/hooks/useCancellableJob.ts`：内部管理 `cancelledRef` 与卸载时取消，导出工厂函数 `jobOptions(onProgress?)` 返回 `PollJobOptions`。
- 五个组件改用该 hook；`DataPanel` 备份导出改走 `pollJobResult`，删除本地 `pollBackupResult` 与 `delay`。

### 验收
- 各组件功能不变，卸载时仍能停止轮询。
- `npm run build` 通过，无重复样板。

## P1-B purge 分项回执 UI

### 现状
- 清空后仅在 status 行显示总数，看不到各表明细。

### 方案
- `DataPanel` 新增 `purgeReport` 状态，清空成功后展示 `deleted_rows` 各表分项 + `removed_files` + 是否含设置。
- 新增轻量回执卡片样式。

### 验收
- 清空后显示“本次清空报告”，列出每表删除行数与文件数。
- `npm run build` 通过。

## 执行顺序
P1-A → P1-B → 验证（build + pytest 确认未回归）→ 升版本 0.1.5 + release notes + README → 提交并打 tag 自动发版。
